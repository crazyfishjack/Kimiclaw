# AI 晨报存档

## 用途
本文件夹用于存档每日 AI 晨报推送的原文记录。

每次向用户推送微信晨报消息时，同步将当日内容存档于此，方便后续检索和回顾。

---

## 命名规则
- 文件名格式：`YYYY年MM月DD日_AI晨报.md`
- 示例：`2026年06月28日_AI晨报.md`
- 语言：中文命名，含日期全称

## 内容格式
每篇存档文件包含：
1. **推送日期** — 文件顶部标注
2. **内容摘要** — 当日推送的全部内容（论文、项目、文章等）
3. **原文链接** — 每条信息附带的 URL
4. **来源标注** — arXiv / GitHub / 稀土掘金 / 微信公众号等

---

## 更新规则
- 每日推送后自动创建新文件
- 不覆盖历史文件
- 永久保留，作为个人知识库

---

## 设计文档 (DESIGN.md)

本文件夹下包含 `DESIGN.md`，详细记录了 AI 晨报推送机器人的完整设计：

- **系统架构** — 抓取 → 筛选 → 总结 → 推送 → 存档的全流程
- **数据抓取模块** — arXiv、GitHub Trending、稀土掘金、微信公众号等
- **质量筛选与去重** — 评分算法、永久去重机制
- **中文结构化总结** — 基于原文内容生成【核心问题】【方法】【结果/意义】
- **GitHub 存档机制** — 自动生成 Markdown 文件并推送到仓库
- **推送机制** — 自动推送（HEARTBEAT）与手动触发
- **已知问题与待优化** — 当前问题和未来改进方向
- **部署与运维** — 文件结构、运行命令、定时任务配置
- **用户交互记录** — 关键决策日志和用户偏好

### 如何维护 DESIGN.md

**触发更新时机：**
1. 新增/移除数据源时
2. 修改总结格式或算法时
3. 调整推送机制时
4. 发现新的待优化问题并解决后
5. 用户提出新的功能需求并实现后

**更新流程：**
1. 在本地修改 `AI晨报存档/DESIGN.md`
2. 更新版本号（如 v1.0 → v1.1）
3. 在"用户交互记录"章节追加新的决策日志
4. 在"已知问题与待优化"章节更新状态
5. `git add` → `git commit` → `git push`

**注意：**
- 每次提交需写明变更内容，如 `docs: 更新设计文档 v1.1 - 新增XXX数据源`
- 保持文档与实际代码的一致性
- 用户可随时要求查看或更新设计文档
